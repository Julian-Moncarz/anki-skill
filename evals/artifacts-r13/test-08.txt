**TCP (Transmission Control Protocol)** establishes a connection before sending data and guarantees reliable, ordered delivery. It retransmits lost packets and uses flow control. Used for HTTP, email, file transfers — anything where data must arrive complete and in order.

**UDP (User Datagram Protocol)** is connectionless and "fire-and-forget." No delivery guarantee, no ordering, no retransmission. It's faster and lower overhead. Used for video streaming, DNS lookups, online gaming — where speed matters more than perfect delivery.

| | TCP | UDP |
|---|---|---|
| Connection | Connection-oriented (3-way handshake) | Connectionless |
| Reliability | Guaranteed delivery, retransmits | No guarantee |
| Ordering | Preserves order | No ordering |
| Speed | Slower (overhead) | Faster |
| Use cases | Web, email, file transfer | Streaming, DNS, gaming |
