**TCP (Transmission Control Protocol)** establishes a connection before sending data and guarantees reliable, ordered delivery. It retransmits lost packets and uses flow control. Used for web browsing (HTTP), email, file transfers — anything where data must arrive complete and correct.

**UDP (User Datagram Protocol)** is connectionless and "fire-and-forget." No guarantees on delivery, ordering, or duplicate protection, but much lower overhead and latency. Used for video streaming, online gaming, DNS lookups, VoIP — where speed matters more than perfection.

| | TCP | UDP |
|---|---|---|
| Connection | 3-way handshake | None |
| Reliability | Guaranteed delivery | Best-effort |
| Ordering | Preserved | Not guaranteed |
| Speed | Slower (overhead) | Faster |
| Header size | 20+ bytes | 8 bytes |
| Use cases | HTTP, SSH, FTP | DNS, streaming, gaming |

The short version: TCP prioritizes correctness, UDP prioritizes speed.
