That error occurs when you try to iterate over (e.g., loop through, unpack, or pass to `list()`/`tuple()`) a value that is `None` instead of an iterable like a list, dict, or string.

**Common causes:**

1. **A function that doesn't explicitly return a value:**
   ```python
   def get_items():
       items = [1, 2, 3]
       # forgot to return items

   for item in get_items():  # get_items() returns None
       print(item)
   ```

2. **An in-place method that returns `None`:**
   ```python
   items = [3, 1, 2].sort()  # .sort() modifies in-place, returns None
   for item in items:  # TypeError
       print(item)
   ```
   Fix: `items = sorted([3, 1, 2])` or call `.sort()` separately.

3. **Unpacking a `None` return value:**
   ```python
   a, b = some_function()  # fails if some_function() returns None
   ```

4. **A variable that's conditionally assigned:**
   ```python
   results = db.query(...) if condition else None
   for r in results:  # TypeError if condition was False
   ```

**How to fix it:**

- **Find the exact line** from your traceback — what variable is `None`?
- **Trace where that variable gets its value** — is it a function return, a method call, or an assignment?
- **Add a guard** if `None` is a valid state:
  ```python
  for item in (items or []):
      ...
  ```
