**TCP (Transmission Control Protocol)** and **UDP (User Datagram Protocol)** are both transport-layer protocols, but they differ in key ways:

| Feature | TCP | UDP |
|---|---|---|
| **Connection** | Connection-oriented (3-way handshake) | Connectionless |
| **Reliability** | Guaranteed delivery with acknowledgments | No delivery guarantee |
| **Ordering** | Packets arrive in order | No ordering guarantee |
| **Error checking** | Yes, with retransmission | Basic checksum only |
| **Speed** | Slower (due to overhead) | Faster (minimal overhead) |
| **Flow control** | Yes (congestion/flow control) | No |
| **Use cases** | Web (HTTP), email, file transfer | Streaming, gaming, DNS, VoIP |

**In short:**
- **TCP** prioritizes **reliability** — it makes sure every byte arrives, in order, even if it takes longer.
- **UDP** prioritizes **speed** — it fires packets off without waiting for confirmation, accepting some loss.

A common analogy: TCP is like a phone call (you establish a connection, confirm the other side is there), while UDP is like sending postcards (you just send them and hope they arrive).
