TCP (Transmission Control Protocol) and UDP (User Datagram Protocol) are both transport layer protocols, but they differ in key ways:

**TCP:**
- **Connection-oriented** — establishes a connection via a three-way handshake before sending data
- **Reliable delivery** — guarantees packets arrive in order, retransmits lost packets
- **Flow control & congestion control** — adjusts sending rate based on network conditions
- **Higher overhead** — headers are 20+ bytes; more latency due to connection setup and acknowledgments
- **Use cases:** web browsing (HTTP/HTTPS), email, file transfers, SSH

**UDP:**
- **Connectionless** — sends datagrams without establishing a connection first
- **No delivery guarantee** — packets can arrive out of order, be duplicated, or get lost with no retransmission
- **No flow/congestion control** — sends at whatever rate the application dictates
- **Lower overhead** — headers are only 8 bytes; minimal latency
- **Use cases:** video streaming, online gaming, DNS lookups, VoIP

**In short:** TCP trades speed for reliability; UDP trades reliability for speed. Choose TCP when data integrity matters, UDP when low latency matters and occasional packet loss is acceptable.
